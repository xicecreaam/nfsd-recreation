# Setting up for speedruns

All the information needed to set up the PC version of Carbon for speedruns.

## Game version

A **1.4 version** of the game is preferred, which can be found very easily on the internet. Alternatively, the **1.2 version** of the game can be an option **but** i did not try to do any FG run so far. It seems that 1.4 is more compatible with modern Windows (10/11) than 1.2. The language does not matter except Chinese (both <ins>Traditional</ins> and <ins>Simplified</ins>), Japanese, Korean, Thai and Russian. Those requires the **1.3 version of the game with their own executable**. Only English, French, Italian, German, Spanish, Russian and Japanese have a dubbing, others use the English one.

### Console ?

Console versions have inconsistent framerates, more loading screens, longer load times and lower accessibility. It would be shooting yourself in the foot running on console.

## Validating your installation

Make sure the intro videos and cutscenes play correctly. Invalid installations with missing or broken videos are not allowed. For example, when you launch the game, after the very short loading screen, if you are automatically sent to the splashscreen (the `Press START to begin` screen), your installation is invalid.

Make sure that the game music, sound effects, assets all play and appear correctly. Modified installations are not allowed. For more information, read [the Game rules from speedrun.com](https://www.speedrun.com/nfsc?h=Career_Any-NG&rules=game&x=xd1olzk8-jlz94y82.jqzg5e4l).

### Collector's Edition

The version of your game has to be the Collector's Edition. It enables additional content and **grants you extra cash when you start a new career (15000 $)**.  
Most of the games wandering on the internet are Collector's Edition since it is the most stable and up to date.

### Widescreen fix

**The only modification currently allowed is [an older version of widescreen fix](https://www.speedrun.com/nfsc/resources/teq52).** Why an older version ? Newer versions contain an setting called `SimRate` that modifies the physics of the game in general. To install widescreen fix, extract the contents of the archive into your game folder.

Unlike Most Wanted, the settings included in the `NFSCarbon.WidescreenFix.ini` work fine. You can set up the game as you want except the `SkipIntro` entry. **SkipIntro isn't allowed for speedruns**, since it lets you skip movies/cutscenes.

> :bangbang: You can set up a custom resolution : **1920x800**, which let you reach further in the world map. In other words, you can start races that should be out of view in 1920x1080 for example.

### Controller support

#### Xbox (XInput)

The game has full support for XInput (Xbox) controllers including button overlays with `ImproveGamepadSupport = 1` in `NFSCarbon.WidescreenFix.ini`.

#### PlayStation

PlayStation controllers require extra steps. The recommended method is using Steam and [adding Carbon as a Non-Steam Game](https://help.steampowered.com/en/faqs/view/4B8B-9697-2338-40EC). This way you can use Steam to configure your controller. To display Playstation buttons ingame, set `ImproveGamepadSupport = 2` in `NFSCarbon.WidescreenFix.ini`.

#### Deadzone

You can configure the deadzone in `NFSCarbon.WidescreenFix.ini` by changing `LeftStickDeadzone`. You can use [Gamepad Tester](https://hardwaretester.com/gamepad) to determine the minimum safe value. Best setting depends on your preference. If you want a sensitive stick, set it as low as you can without the car turning on its own, otherwise use slightly higher values. The default value of 10 works well enough for most.

## Recommended settings

You can **enable compatibilty mode** for Windows 7, it may help for some reasons.

The game can randomly crash, primarily on loading screens, unless you **set affinity to a single core or thread**. To prevent crashes, you have two solutions :

* Method 1

    You can [download](https://drive.google.com/file/d/0B1QRkF64MSUSOTZyS2E3anFBUzA/view?resourcekey=0-FjHxvFlsJS7DF5NWP8rxeg) and place a .bat file next to your game executable that automatically set the process to 1 core/thread and launch the game.

* Method 2

    Alternatively, you can use an application like [Process Lasso](https://bitsum.com/download-process-lasso/) to automatically change the affinity whenever the game starts. Launch the game then right click the process (NFSC.exe) in Process Lasso : CPU Affinity > Always > Select CPU Affinity > Untick ALL, tick a single CPU and click OK. Avoid CPU 0 and 1 if possible, because Windows often uses these for background processes by default.

<span style={{ fontSize: '1.3rem', fontWeight: 'bold' }}>
  Audio
</span>

**Disable speech volume** by setting on `0`. This means a faster tutorial and no police chatter, which prevents cop spawns during races.  
**TURN OFF EA TRAX**. YouTube doesn't recognize that the tracks are licensed for Carbon, and some songs outright block videos worldwide, which would result in your run being rejected.

<span style={{ fontSize: '1.3rem', fontWeight: 'bold' }}>
  Video
</span>

**Shaders to high** for the best visibility (doesn't affect loads).  
**World Details to low or medium** to hit less stuff on the track.  
**Disable Motion Blur** in the Video settings.  
Let the **Particle System on**, it allows you to see the lap/finish markers.  
Set everything else **as high as your PC can handle** for better visibility in general.

If you experience low performance, make sure the game is using your dedicated graphics card. Reduce graphics settings like Shadow Detail and Visual Treatment (like Most Wanted, the blue filter in Carbon can be annoying). You can also try using [dxvk](https://github.com/doitsujin/dxvk/releases), which can increase performance by 20-30% on some modern systems. To use it, place the 32 bits d3d9.dll next to the game executable.

<span style={{ fontSize: '1.3rem', fontWeight: 'bold' }}>
  Gameplay
</span>

Enable Autosave, this way you can reload your savefile easily if a terrority attack happens.  
Set your transmission to **`manual`**. It is faster than `automatic`, it shifts early and reduces top speed. Manual gearbox also allows precise gear control.  
The camera choice is up to you but the preferred one is **hood camera**. Using it, you judge corners better.  
**Disable Game Moment Camera in gameplay options.** It removes the cop zoom-in when you trigger a chase / jump camera when you jump.

<span style={{ fontSize: '1.3rem', fontWeight: 'bold' }}>
  Interface
</span>

**Enable rearview mirror**. With it, you can see rubberbanding cars from behind to potentially block.  
**Disable Picture in Picture (PiP)**, you will get a small timesave in the tutorial.

<span style={{ fontSize: '1.3rem', fontWeight: 'bold' }}>
  Controls
</span>

Bind `Accelerate` and `Brake/Reverse` to your controller triggers, `Shift Down`, `Shift Up` and `Clutch` to accessible locations.

    Other binds like `NOS` or `Reset Car Position` are up to your preference.
  </font>

### Framerate requirements

**Locking the frame rate up to 120 FPS is required for speedruns.** If not, I recommend [RivaTuner Statistics Server (RTSS)](https://www.guru3d.com/download/rtss-rivatuner-statistics-server-download/) or your graphics driver.

**Displaying your framerate is also required.** You can use [RivaTuner Statistics Server (RTSS)](https://www.guru3d.com/download/rtss-rivatuner-statistics-server-download/) for this too, or the overlay of your graphics driver. If you are using [OBS](https://obsproject.com/) and RTSS, enable Use Microsoft Detours API hooking in the settings of RTSS and enable Capture third-party overlays in the Game Capture of OBS.

## LiveSplit and timing

LiveSplit is a timer program for speedrunners that is both easy to use and full of features. You can download it [here](https://livesplit.org/downloads/).

**The primary timing method for this game is Load Removed Time (LRT)**, or LoadLess (LL). Loading times are not consistent in this game, depending on FPS, hardware configuration and loaded content. Since the game generates random racers, the loading time is different every time you enter a race.

**Please use LiveSplit with the load remover and auto-splitter activated in the Splits Editor**. By settting the Game Name to `Need for Speed: Carbon`, the load remover + auto-splitter appears and you will be able to activate and modify it.

**Make sure Refresh Rate in LiveSplit Settings is set to 60 or higher**, otherwise the load remover and autosplitter might not work correctly.

## `Open World Map` macro

Instead of spamming your `Open World Map` keybind, you can use a macro using AutoHotkey. AutoHotkey is a program that allows users to easily create small to complex scripts for all kinds of tasks. You can download it [here](https://www.autohotkey.com/download/ahk-v2.exe).

Two types of scripts are available :

* a controller script : the `Joy4` keybind
> * for XInput controllers → `Y` button
> * for DualShock controllers → `Trackpad` button
* a keyboard script : the `j` keybind

[**Download macros here**](https://mega.nz/folder/pswBUYJS#7bGNU9nc3560UJemv-nchg). **Please note that only these macros are permitted. Any other use of scripts or macros is prohibited based on the [rules of speedrun.com](https://www.speedrun.com/support/learn/moderation-rules)**
